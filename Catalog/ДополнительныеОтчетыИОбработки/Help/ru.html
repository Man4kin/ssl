В зависимости от назначения, обработки подразделяются на следующие виды:

  • Обработки заполнение объекта
  • Обработки создания объектов на основании
  • Печатные формы
  • Отчеты
  • Дополнительные отчеты
  • Дополнительные обработки

При этом первые четыре вида обработок используются в различных формах объектов (т.н. назначаемые обработки), а два последних вида вызываются отдельно в командном интерфейсе. Для назначаемых обработок необходимо указать типы объектов, с которыми они должны работать.

Каждая обработка может поставлять одну или несколько команд.

Технически, дополнительные отчеты и обработки могут быть вызваны из конфигурации тремя способами:

  • Открытие формы - при этом открывается форма обработки
  • Вызов клиентского метода - при этом вызывается метод в клиентском контексте
  • Вызов серверного метода - при этом вызывается метод в серверном контексте

 

Варианты публикации применяются для включения или отключения отдельных обработок:

  • Используется - обработка доступна для использования
  • Не используется - обработка не доступна для использования
  • Режим отладки - обработка доступна для использования только пользователю с ролью "Добавление и изменение дополнительных обработок" или ролью "Полные права"

 

Для серверных команд дополнительных (не назначаемых) обработок также возможно указать расписание для выполнения по регламентному заданию.
